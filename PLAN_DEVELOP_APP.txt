Lineamientos escenciales
- Aplicacion de escritorio multiplataforma (Windows/macOS/Linux)
- Usar siempre buenas practicas
- Proveer de atajos desde el teclado, el usuario debe poder manejar la aplicacion sin usar el mouse

Funcionalidades:
- 100% offline
- Capacidad de sincronizar datos con un servidor cuando hay conexion a internet
- Capadidad de leer inputs por dispositivos externos como maquina lectora de códigos de barras
- Servidor para sincronizacion de datos
- API para comunicacion entre la aplicacion y el servidor


Infraestructura:
- Base de datos local (los productos tienen que estar respaldados, para ante cualquier error nunca perder los datos)
- Tablas:
    - usuarios (id_usuario, nombre, password, rol)
    - productos (id_producto, nombre, categoria, precio, stock, codigo de barras, descripcion, id_proveedor)
    - proveedores (id_proveedor, nombre, telefono, direccion, email)
    - ventas (id_venta, id_producto, cantidad, precio, fecha, id_usuario)
    - egresos (id_egreso, descripcion, monto, fecha, id_usuario)
    - logs (id_log, descripcion, fecha, id_usuario)


Packages
Frontend (Node.js / npm)
Estos son los que instalarás en la raíz de tu proyecto para manejar la interfaz y la lógica de negocio.
@tauri-apps/api: Imprescindible. Es el puente para que tu React hable con el sistema operativo (ventanas, archivos, hardware).
@tauri-apps/plugin-sql: El plugin oficial de Tauri v2 para manejar SQLite de forma nativa y segura.
drizzle-orm: Para manejar la base de datos con TypeScript de forma fluida.
drizzle-kit: (Dev dependency) Para generar las migraciones (archivos .sql) automáticamente cuando cambies tus tablas.
zustand: Mucho más liviano y simple que Redux para manejar el estado global (el carrito de compras actual, el usuario logueado).
shadcn/ui + tailwind-merge: Para la interfaz. Te da componentes profesionales (tablas, diálogos, inputs) que transmiten ese "orden" que buscás.
lucide-react: El paquete de iconos estándar, limpio y consistente.
zod: Para validar los datos que entran (ej. que un precio no sea negativo) antes de guardarlos en la DB.

Bash
# Comando de instalación rápida
npm install @tauri-apps/api @tauri-apps/plugin-sql drizzle-orm zustand lucide-react zod
npm install -D drizzle-kit tailwindcss postcss autoprefixer

2. Backend (Rust / Cargo)
Estos van dentro de src-tauri/Cargo.toml. Tauri v2 maneja casi todo por plugins, pero necesitás estos para el hardware:
tauri-plugin-sql: El lado de Rust del plugin de base de datos.
escposify o escpos: Crates de Rust para convertir texto y logos al lenguaje de las impresoras térmicas (comandos hexadecimales).
serde y serde_json: Para convertir los datos que viajan entre Rust y TypeScript de forma segura.
chrono: Para un manejo preciso de fechas y horas en los cierres de caja (evita líos de zonas horarias).

centauri-balance/
├── src/                # Frontend (React + TS)
│   ├── assets/         # Imágenes, logos, sonidos de escaneo
│   ├── components/     # UI genérica (Buttons, Inputs, Modals)

│   ├── hooks/          # useScanner.ts, usePrinter.ts, useDatabase.ts
│   ├── lib/            # Configuración de Drizzle, clientes
│   ├── services/       # Llamadas a Tauri (invokes)
│   ├── store/          # Estado global (Zustand o Context)
│   └── types/          # Interfaces compartidas
├── src-tauri/          # Backend (Rust)
│   ├── capabilities/   # Permisos de Tauri v2 (impresión, fs, etc.)
│   ├── migrations/     # Archivos .sql de Drizzle
│   ├── src/
│   │   ├── commands/   # Funciones de Rust (printer.rs, db.rs, auth.rs)
│   │   ├── main.rs     # Punto de entrada y registro de comandos
│   │   └── setup.rs    # Inicialización de la DB y carpetas
│   └── tauri.conf.json # Configuración nativa
└── drizzle.config.ts   # Configuración de migraciones
